## Purpose
Get structured, honest critique on any piece of design work — UI, brand, layout, or visual identity.

## Load Skills
> Before proceeding, read and apply: `skills/design/SKILL.md`

## Prompt

```
Critique the following design work.

What this design is for: [PURPOSE AND CONTEXT]
Intended audience: [WHO IT'S DESIGNED FOR]
Stage of work: [EARLY CONCEPT / MID DEVELOPMENT / NEAR FINAL]
What I'm most uncertain about: [YOUR SPECIFIC CONCERN OR QUESTION]
What feedback I need: [HIGH-LEVEL DIRECTION / DETAILED NOTES / BOTH]

[PASTE DESCRIPTION, SHARE IMAGE, OR DESCRIBE THE WORK IN DETAIL]
```

## Usage Notes
- Attach or describe the work as specifically as possible — vague descriptions get vague feedback
- Stating what you're uncertain about focuses the critique on what's actually useful
- "Stage of work" matters: early concepts need directional feedback, near-final work needs detail
- For UI work, describe the user flow or interaction model, not just the visual
