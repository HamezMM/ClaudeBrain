## Purpose
Write a thorough design brief for any creative project — branding, UI, print, campaign, or campaign assets.

## Load Skills
> Before proceeding, read and apply: `skills/design/SKILL.md`

## Prompt

```
Write a design brief for the following project.

Project type: [brand identity / UI design / print / campaign / motion / other]
Client or context: [WHO IS THIS FOR — COMPANY, PRODUCT, EVENT]
What this design must achieve: [THE GOAL IN ONE SENTENCE]
Audience: [WHO WILL SEE OR USE THIS]
Tone and feeling: [HOW SHOULD IT FEEL — E.G. BOLD, TRUSTWORTHY, PLAYFUL, PREMIUM]
Constraints: [BRAND RULES, TECHNICAL LIMITS, BUDGET, TIMELINE]
What to avoid: [STYLES, REFERENCES, OR DIRECTIONS TO STAY AWAY FROM]
References I like: [LINKS, NAMES, OR DESCRIPTIONS OF WORK I ADMIRE]
Deliverables needed: [WHAT EXACTLY MUST BE PRODUCED]
```

## Usage Notes
- Completing every field produces a significantly better brief — don't skip constraints or "what to avoid"
- Paste in any existing brand guidelines or style notes alongside the prompt
- Ask Claude to generate reference directions ("moodboard concepts") if you're still deciding on tone
